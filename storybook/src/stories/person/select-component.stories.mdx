# PersonSelect

## Properties/Attributes

Name                      | Type                        | Default          | Description
------------------------- | -------------------------   | --------------   | -----------
`selectedPerson`          | `PersonInfo - string(upn - azureId) - null` | `undefined` | Select the defined person or clear any selected person by setting null as value. see [examples](?path=/docs/examples-person-person-select--component)
`placeholder`             | `string`                    | `undefined`      | TextInput placeholder string
`variant`                 | `variants*`                 | `page`           | The type of Searchabledropdown to render, multiple variants for header or page usage
`dropdownHeight`          | `string`                    | `undefined`      | The scroll height for dropdown result list
`leadingIcon`             | `string`                    | `undefined`      | Leading Icon to display in text input
`autofocus`               | `boolean`                   | `false`          | Flag for setting focus on textInput on component mount
`disabled`                | `boolean`                   | `false`          | Disable text input
`label`                   | `string`                    | `undefined`      | Text input label
`value`                   | `string`                    | `undefined`      | The selected value do display in TextInput

---
\*  available `variants` values are:
```ts
'page' | 'page-outlined' | 'page-dense' | 'header' | 'header-filled';
```

## Example Usage

```tsx
import { PersonSelect } from '@equinor/fusion-react-person';

<PersonSelect placeholder="Start to type to search..."
  initialText="Start to type to search..."
  variant="page"
  dropdownHeight="300px"
  leadingIcon="search" />
```

## PersonProvider

The PersonProvider is provided by the Portal or CLI.

You can define a custom [**PersonProvider**](?path=/docs/data-person-provider--page) wrapper for your PersonSearch component.


## Controlled component - selectedPerson

To set selected person in component use the `selectedPerson` attribute.

You can select a person by supplying a persons `azureId` or `upn` (email) address or a complete `PersonInfo` object.

To clear a selected person, supply `null` as value. This will also clear any future manual selection from the interface.

Remove the attribute to return to a uncontrolled component state.

see [examples](?path=/docs/examples-person-person-select--component).
