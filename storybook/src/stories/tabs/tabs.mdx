## Properties/Attributes for single Tab
The `TabProvider.Tab` component is used to define individual tabs within the `TabProvider`.

| Name         | Type     | Default | Description                                                                                                                     |
|--------------|----------|---------|---------------------------------------------------------------------------------------------------------------------------------|
| id           | string   | /       | A unique key used to identify the tab within the tab group. This key is also used when seting active tab and for building URL structure for showing selected tab.   |
| title        | string   | /       | The text displayed in the tab header.                                        |
| right        | boolean  | false   | If set to true, this tab will be visually aligned to the right side of the tab header area.     |

## Controlled Tabs
This example uses the `useActiveTab` hook to control which tab is currently active. 
In this case, each tab contains a button that switches the tab.

Useful for:
- Custom navigation controls inside tab content.
- Conditional logic for navigating between steps/tabs.

Important Notes:
- `useActiveTab` must be called **inside a component thatâ€™s rendered within TabProvider**.
- You can use it to both read and change the active tab with tab id or index:
  - **activeTab**, 
  - **setActiveTab**, 
  - **setActiveTabIndex**
