## How to use

The ``searchable-dropdown`` component is built up of 2 elements, ``dropdown`` and ``dropdownProvider``.

Wrap the dropdown component in the provider and add a ref to the provider element by using the hook ``useDropdownProviderRef`` with the resolver as parameter.
The resolver must be of type ``SearchableDropdownResolver``.

The resolver has the method ``searchQuery`` which is called on every keyup event with 250ms debounce from the textinput element. So place your API call on the resolvers searchQuery method.
Your API call should return an array of type ``SearchabledropdownResult`` where each item is of type ``SearchableDropdownResultItem`` with the following properties.

```typescript
interface SearchableDropdownResultItem {
id: string;
  title?: string;
  subTitle?: string;
  isDisabled?: boolean;
  isSelected?: boolean;
  isError?: boolean;
  meta?: string;
  graphic?: string;
  type?: 'section' | 'divider' | null;
  children?: SearchableDropdownResult;
}
```

When the user selects and item from the result list the ``onSelect`` event gets triggered. The ``onSelect`` events detail object contains the selected item, or items if multiple flag is set. 

Listen for the ``onSelect`` event and save the selected item in your context/state handler.

## Composition

SearchableDropdown consist of to components and a hook to attach a resolver to the controller.

The components are ``Dropdown`` and ``DropdownProvider``, And the hook ``useDropdownProviderRef``.

Se [Examples](?path=/docs/examples-searchable-dropdown--seperated-components) for usage.
